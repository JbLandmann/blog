## 2026-03-20 — Architecture Deepening: Content Pipeline, Components, SEO Engine

**Trigger**: `improve-codebase-architecture` skill analysis identified 3 friction points in the ~760 LOC blog: duplicated draft filtering (3×), date formatting (4×), tag rendering (2×), and 53-line SEO `<head>` soup inlined in Base.astro.

**What changed**:
- Created `src/lib/content.ts` — Functional Toolkit with `getPublishedPosts()`, `sortByDateDesc()`, `formatDate()`, `postUrl()`. Draft filtering now has a single enforcement point.
- Created `src/components/` — 3 Atomic Primitives: `FormattedDate.astro`, `TagList.astro`, `CoverImage.astro`. Eliminates all markup duplication.
- Created `src/lib/seo/` — Headless Spec pattern: `resolvePageSeo()` pure function → `ResolvedSeo` typed object → `<Seo />` Astro component. Base.astro shrank from 152 to 55 lines.
- Created `src/__mocks__/astro-content.ts` + updated `vitest.config.ts` to alias `astro:content` for unit testing.
- Added 30 new unit tests (10 content + 20 SEO), all pure function tests running without Astro build.

**Impact**: Test count 18 → 48. All logic now testable in isolation. Duplication eliminated. SEO metadata fully configurable via `DEFAULT_CONFIG`.

### 2026-03-23 — PostCard deepening: extract card composition into reusable component

**Trigger**: Architecture analysis (improve-codebase-architecture skill) identified the "post card" concept scattered across 3 micro-components + 24 lines of inline rendering in index.astro — no reuse boundary, untestable composition.

**Changes**:
- Created `src/components/PostCard.astro` — deep component owning full card rendering (image, date, title, description, tags, link, hover). Interface: `{post: BlogPost, class?: string}`.
- Simplified `src/pages/index.astro` — replaced 24-line inline composition with `<PostCard post={post} />`. Removed 3 direct component imports (FormattedDate, CoverImage, TagList) + postUrl.
- Created GitHub issue [#2](https://github.com/JbLandmann/blog/issues/2) documenting the architectural rationale.

**Impact**: Future features (F05 reading time, F11 tag pages, F12 related articles, F15 series) can now reuse PostCard directly instead of copy-pasting card rendering. index.astro reduced from 57 to 38 lines.
